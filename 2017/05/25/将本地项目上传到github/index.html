将本地项目上传到github

发表于 2017-05-25   |   分类于 工具

  (≧▽≦)/~啦啦啦，端午节快乐！我们首先要注册一个github账户，并设置好ssh。这里我就默认这些大家已经装好，然后就开始上传项目倒GitHub吧！

新建项目

把文件夹变成Git可管理的仓库

git init


这个时候newProject文件夹下面会有一个.git的文件夹，它是Git用来跟踪和管理版本库的。这个文件夹是影藏文件，你需要设置一下让隐藏文件可见。然后写上你的项目的内容。

将文件添加到git上

git add
git commit -m ‘注释’

github新建项目


记住这个项目的GitHub地址，下图红色框里的内容。

本地文件个github项目关联

git remote add origin https: //github.com/whalealice/newProject.git
git push -u origin master


执行完成之后如果出现上图一样的报错，原因是github中的README.md文件不在本地代码目录中。
可以通过如下命令进行代码合并【注：pull=fetch+merge]

git pull –rebase origin master
这是文件夹内会出现README.md文件。

此时再执行语句 git push -u origin master即可完成代码上传到github。
